# 修复 401 (Unauthorized) 错误

如果您遇到 `401 (Unauthorized)` 错误，这是因为 Google Apps Script Web App 的权限设置不正确。请按照以下步骤修复：

## 步骤 1：检查并修复 Web App 权限设置

### 方法 A：通过部署管理界面（推荐）

1. **打开 Google Apps Script 编辑器**
   - 打开您的 Google Sheet
   - 点击 **扩展程序** > **Apps Script**

2. **进入部署管理**
   - 点击 **部署** > **管理部署**
   - 找到您的 Web App 部署
   - 点击部署旁边的 **编辑**（铅笔图标 ⚙️）

3. **修改权限设置**
   - 在 **具有访问权限的用户** 下拉菜单中
   - **必须选择：任何人**（这是关键！）
   - **不要选择**："只有我自己" 或 "需要登录"

4. **创建新版本**
   - 在 **版本** 下拉菜单中，选择 **新版本**
   - 点击 **部署**

5. **复制新的 URL**
   - 部署完成后，复制新的 **Web 应用 URL**
   - 如果 URL 有变化，更新 `index.html` 中的 `GOOGLE_SCRIPT_URL`

### 方法 B：删除并重新部署

如果方法 A 不起作用，可以删除旧部署并创建新部署：

1. **删除旧部署**
   - 在 **部署** > **管理部署** 中
   - 点击部署旁边的 **删除**（垃圾桶图标）
   - 确认删除

2. **创建新部署**
   - 点击 **部署** > **新建部署**
   - 点击 **选择类型** 旁边的齿轮图标 ⚙️
   - 选择 **网页应用**

3. **设置部署选项**
   - **说明**：Vibe Engineering Registration
   - **执行身份**：选择 **我**
   - **具有访问权限的用户**：**必须选择 "任何人"** ⚠️
   - 点击 **部署**

4. **首次授权**
   - 点击 **授权访问**
   - 选择您的 Google 账号
   - 点击 **高级** > **前往 [项目名称]（不安全）**
   - 点击 **允许**

5. **复制 Web App URL**
   - 复制显示的 **Web 应用 URL**
   - 更新 `index.html` 中的 `GOOGLE_SCRIPT_URL`

## 步骤 2：验证权限设置

1. **测试 Web App**
   - 在浏览器中打开您的 Web App URL
   - 如果看到 `{"status":"OK","message":"Google Apps Script 已就绪"}`，说明权限正确
   - 如果看到登录页面或错误，说明权限设置仍有问题

2. **使用匿名/无痕窗口测试**
   - 打开浏览器的无痕/隐私模式
   - 访问 Web App URL
   - 如果不需要登录就能访问，说明权限设置正确

## 步骤 3：更新代码（如果需要）

我已经更新了代码，使用 iframe 方法来提交表单，这样可以更好地处理权限和 CORS 问题。

如果您的 Web App URL 有变化，请更新 `index.html` 中的这一行：

```javascript
const GOOGLE_SCRIPT_URL = '您的_Web_App_URL_这里';
```

## 步骤 4：测试表单提交

1. 打开 `index.html` 页面
2. 填写表单并提交
3. 检查 Google Sheet 中是否出现新数据
4. 如果仍有问题，检查浏览器控制台（F12）中的错误信息

## 常见问题

### Q: 为什么必须设置为"任何人"？
A: 因为表单是从网页提交的，用户可能没有登录 Google 账号。设置为"任何人"允许匿名提交。

### Q: 这样安全吗？
A: Google Apps Script 代码仍然在您的账号下运行，只有您授权的代码可以访问 Sheet。设置为"任何人"只是允许调用 Web App，不会暴露您的 Sheet 数据。

### Q: 如果我不想设置为"任何人"怎么办？
A: 如果必须限制访问，您需要实现自己的认证系统，或者使用 Google OAuth。但对于简单的报名表单，"任何人"是最实用的选择。

## 如果问题仍然存在

1. **检查执行日志**
   - 在 Apps Script 编辑器中，点击 **查看** > **执行日志**
   - 查看是否有错误信息

2. **检查 Sheet 权限**
   - 确认 Google Sheet 本身没有限制访问
   - Sheet 应该至少对您的账号可编辑

3. **重新部署**
   - 有时需要完全删除旧部署并创建新部署
   - 确保每次修改代码后都创建"新版本"

4. **检查 URL**
   - 确认 URL 以 `/exec` 结尾（不是 `/dev`）
   - `/dev` 版本需要登录，`/exec` 版本是已部署的版本

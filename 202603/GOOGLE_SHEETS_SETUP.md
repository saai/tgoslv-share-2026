# 202603 活动报名 · 新建 Google Sheet 配置步骤

202603 使用**独立的** Google Sheet 和 Web App，表格包含「活动」列（可填写如 202603 或留空）。按下列步骤操作即可。

---

## 步骤 1：新建 Google Sheet

1. 打开 [Google 云端硬盘](https://drive.google.com)，点击 **新建** → **Google 表格**。
2. 给表格起名（例如：`TGO 202603 活动报名`）。
3. 在**第一行**填写表头（共 5 列）：

   | A 列 | B 列 | C 列 | D 列 | E 列     |
   |------|------|------|------|----------|
   | 活动 | 姓名 | 邮箱 | 公司 | 提交时间 |

4. 复制该 Sheet 的 **ID**：
   - 打开浏览器地址栏，URL 形如：`https://docs.google.com/spreadsheets/d/这里是一串字母数字/edit`
   - `d/` 和 `/edit` 之间的那串就是 **Sheet ID**，复制保存。

---

## 步骤 2：绑定 Apps Script 并粘贴代码

1. 在该 Google Sheet 中，菜单栏选择 **扩展程序** → **Apps Script**。
2. 若提示命名项目，可输入：`202603 报名`。
3. 删除编辑器中默认的 `function myFunction() { ... }` 等代码。
4. 打开本地的 **`202603/google-apps-script.js`**，复制**全部**内容，粘贴到 Apps Script 编辑器。
5. 在代码顶部找到这一行：
   ```javascript
   const SPREADSHEET_ID = 'YOUR_202603_SPREADSHEET_ID_HERE';
   ```
   将 `YOUR_202603_SPREADSHEET_ID_HERE` 替换为您在步骤 1 复制的 **Sheet ID**（保留引号）。
6. 点击 **保存**（或 `Ctrl+S` / `Cmd+S`）。

---

## 步骤 3：部署为 Web 应用

1. 在 Apps Script 编辑器中，点击 **部署** → **新建部署**。
2. 点击 **选择类型** 旁的齿轮图标，选择 **网页应用**。
3. 设置：
   - **说明**：可填 `202603 报名表单`
   - **执行身份**：**我**
   - **具有访问权限的用户**：**任何人**（必选，否则页面提交会报 401/403）
4. 点击 **部署**。
5. 若出现「需要授权」：
   - 点击 **授权访问** → 选择您的 Google 账号 → **高级** → **前往 xxx（不安全）** → **允许**。
6. 部署完成后，在弹窗中复制 **Web 应用 URL**（以 `/exec` 结尾），保存下来。

---

## 步骤 4：把 Web 应用 URL 填进 202603 页面

1. 用编辑器打开 **`202603/index.html`**。
2. 搜索：`YOUR_202603_WEB_APP_URL_HERE`。
3. 将该占位符**整体**替换为步骤 3 复制的 **Web 应用 URL**（保留引号），例如：
   ```javascript
   const GOOGLE_SCRIPT_URL = 'https://script.google.com/macros/s/AKfycby.../exec';
   ```
4. 保存文件。

---

## 步骤 5：测试

1. 在浏览器中打开 **`202603/index.html`**（或您部署该页面的地址）。
2. 填写报名信息并点击 **提交报名**。
3. 打开对应的 Google Sheet，应能看到新一行数据，且 **A 列「活动」** 会显示表单里传的 `202603`（本页面表单已带隐藏字段 `event=202603`）。

---

## 表头说明

- 表格共 5 列：**活动、姓名、邮箱、公司、提交时间**（活动在第一列）。
- 「活动」列由脚本根据表单的 `event` 自动写入（本活动为 `202603`），便于后续筛选或区分多场活动。

---

## 常见问题

- **401 / 403 错误**：在 **部署** → **管理部署** 中确认「具有访问权限的用户」为 **任何人**，并重新部署一次（新版本）。
- **Sheet 里没有新数据**：检查脚本里的 `SPREADSHEET_ID` 是否与当前 Sheet 的 ID 一致；在 Apps Script 中 **查看** → **执行日志** 看是否有报错。
- **修改脚本或 Sheet ID 后**：需在 **部署** → **管理部署** 中选 **新版本** 再部署，否则不会生效。

完成以上步骤后，202603 的报名就会写入您新建的 Sheet，并保留「活动」列。
